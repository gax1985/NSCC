    1. Modern Web Application Structure
      Reference: https://hackr.io/blog/web-application-architecture-definition-models-types-and-more



      A simple overview of a modern day web application (webpage/mobile app). On the client side, users can see the rendering of the content and interact with it via user interfaces such as dropdown menu, buttons, textfield...etc. This is usually done via JavaScript (in browsers/web view on a webapp). The data exchange between the client and the server is usually done via HTML (static content), AJAX (request from client to server), and JSON (usually for data interchange). It's worth to note that with the almost defacto requirement on the use of HTTPS, the communication between the client and server is usually encrypted.

      On the server side, a modern day web app usually requires more than just a static html page. The web server usually connects with database and other related credential validation services and backend. 

      It is obvious that security issues may present in any part in this stack. In this course we will learn how different security vulnerabilities in different part of the system. 
    2. History of Software Security
      As you may have learned from other courses, hacking started whenever security measures were put into place. Put the physical hacking (e.g. lock picking, social engineering..etc) aside, the Enigma Machine can be a good starting point for the discussion of "software" hacking.

      The Enigma Machine was used in WWII and by changing its configuration, the code can be encrypted differently using the same machine, which shares a lot of similarities on modern day "software" (except the configuration was done by changing the physical (mechanical) arrangement, so it is still on the hardware level).

      Another important aspect of the Enigma Machine is the code breaking process by the alliance during WWII. Details can be found from the book, but in short another machine and algorithm was being used for breaking the code, i.e. automated enigma code cracking (~1940). The bombe machine was first built by the Hungarians and latter on the idea was further developed by Alan Turing and the team of researchers in the Bletchley Park in the UK. This machine crack the code by running 20,000 possible machine configurations in around 20 minutes. It is known as known plaintext attack (KPA).

      As technology advanced, different ways of accessing technologies through security flaws and their countermeasures became a routine plot. The phone "phreaking" (~1950)- "anti-phreaking technology through DTMF" (~1960) is a classic example before the age of computer. The first convicted hacker in the US- Robert Morris released the first virus outside of research lab, a "worm" that will infect computers through many flaws which are still popular today, including software flaw, weak password and configuration errors. Learn more about the Morris worm.

      2000s was considered to be the age of the rise of the World Wide Web (WWW), with the widespread use of website and browser technology. Instead of distributing virus and physical hacking into systems, www and the internet provided means for different types of attacks. One critical point was the use of web technology for mission critical tasks to be done on general computers (e.g. e-banking).

      Web (application) security is only one facet in cybersecurity, and in this course we will focus on this widespread, and potentially extremely costly facet. 
    3. Resource we will be using in this course
      We will mainly be using 3 different resources in this course
      • Textbook: Web Application Security by Andrew Hoffman
        Most of the quizzes are related to the content from this book. It will be used to serve as the structure of this course, and each week we will go over part of it together, and discuss related examples and their connection with the OWASP top 10 list.
      • OWASP top 10 list of web application vulnerabilities (2017 & 2021) & Hacksplaining.com
        The OWASP Top 10 list of security issues is based on consensus among the developer community of the top security risks. We will learn about different categories of the 2017 and 2021 version, and compare how things changed over the years and their impacts. 
        There are 5 assignments in this course, all of them are related to the OWASP top 10 list. 
        Hacksplaining is a website with a lot of useful tutorials and information about different hacking approaches and the mitigation and prevention of them. There is a OWASP section on hacksplaining and the assignments were designed to work with these tutorials.
      • Juice Shop - an insecure web application designed for learning different kinds of security issues
        Juice Shop is an application based on the popular Node.js technology, so it is largely based on JavaScript. We will learn how to install it [Installation note- refer to the Local Install Section] (and to understand a Web Application Structure) and this will be used throughout the course and will be referenced in the assignments.
