Injection

In 2023, the order of the top 10 vulnerabilities have remained the same as in 2021.

Injection describes an attack using unintended use of request, commands and code to the original version and obtain control of remote system to change access control and create further data and security breach in the system.
In the Web Application Security book, the author group the attacks to two main categories: Code Injection (including SQL injection) and Command Injection (elevated form of code injection, targeting the host OS). A comprehension table can also be found on IBM's webpage (https://www.ibm.com/docs/en/snips/4.6.0?topic=categories-injection-attacks) listing common injection attacks.
Example on PHP + MySQL code: https://www.w3schools.com/php/php_mysql_select.asp



SQL injection was illustrated in this demo. By including a single quote in the email, we managed to trigger an error message which was fully reported in the http response (reference: when testing sql injection why do testers frequently use single quote). It shown how the user's input (email + password) were being included in a formatted SQL query in the backend. Through including "OR true" and the double dash "--" to comment out the rest of the query. The first user in the database was being returned in the SQL query in the login procedure. This gave us admin access.

This example also specifically demonstrated why SQL injection is one of the most commonly used injection technique. PHP, HTML form and SQL are usually used together, many older implementation of PHP sql query were not validated or scrutinized properly before being incorporated in a database query, and create a system with vulnerability. 

Broken Access Control

As the name suggested, broken access control attacks target the access control mechanism of the system. Access control exists in almost all systems with more than one level of access right. For instance, in BrightSpace, a user can be an instructor or a student in a particular course.  Broken access control allows attacker to access the system as another user (similar access rights) or even an administrator (with higher access rights).

When a user gain unauthorized access through a system flaw and act as another user with same level of permissions, it is referred to as a "horizontal privilege escalation". Using Brightspace as an example, if you have access to your classmate's account without proper authorization, this can be an example of horizontal privilege escalation. 

In contrast, the "vertical privilege escalation" refers to escalation of user's access level to a different level, usually to higher levels or access to different part of the system.

One general principal that can help preventing this kind of attack is the principle of least privilege/deny by default. i.e. if there's no reason to give access, deny by default.

An example of Broken Access Control (Facebook, 2015)- https://thezerohack.com/hacking-facebook-pages

Cryptographic Failure

Previously known as Sensitive Data Exposure in the OWASP Top 10 (2017), Cryptographic failure refers to any systems failed to protect sensitive data via (or the lack of) cryptography.  A classic example of the lack of cryptography protection that caused data leakage will be the use of http instead of https, which allow sensitive information such as username and password to be transmitted in plain text and read by any third party in the transmission process to retrieve. 

Other examples including the use of older, and publicly compromised encryption method such as MD5 or SHA1 hash function. (Reference: Intro to Hashing, MD5 and SHA1)
